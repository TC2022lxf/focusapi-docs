视频生成通常比文本和图片更耗时，很多模型采用 **异步任务**：先提交任务，拿到 `task_id`，再轮询任务状态，最后下载视频结果。

## 支持的模型来源

| 来源 | 适合场景 | 调用特点 |
| --- | --- | --- |
| Seedance | 短视频、运镜、图生视频 | 常见为异步任务，可能有比例、时长、运镜参数 |
| 可灵 / Kling | 图生视频、人物动作、风格视频 | 参数和任务状态以模型广场说明为准 |
| Runway / 其他视频模型 | 创意视频、素材生成 | 可能使用厂商原生参数 |

## 通用接入流程

<Steps>
  <Step title="选择视频模型">
    在模型广场筛选视频能力，复制完整 `model` ID，并确认支持文生视频还是图生视频。
  </Step>
  <Step title="提交任务">
    传入 prompt、图片 URL 或 base64、比例、时长等参数。成功后通常返回 `task_id`。
  </Step>
  <Step title="轮询状态">
    按接口返回的任务 ID 查询状态，直到成功、失败或超时。
  </Step>
  <Step title="获取结果">
    成功后读取视频 URL。注意 URL 可能有有效期，建议及时转存。
  </Step>
</Steps>

## 参数关注点

| 参数 | 说明 |
| --- | --- |
| `model` | 视频模型 ID |
| `prompt` | 视频描述，建议写清主体、动作、镜头、风格 |
| `image` / `image_url` | 图生视频时的首帧或参考图 |
| `duration` | 视频时长，常见为 5s、10s 等模型限定值 |
| `aspect_ratio` / `size` | 画幅比例，如 16:9、9:16、1:1 |
| `seed` | 部分模型支持固定随机种子 |
| `camera` / `motion` | 部分模型支持运镜或运动强度 |

<Note>
  视频模型的接口差异很大。本文档不把每个厂商参数硬塞到一个表里，避免误导；具体可用参数以模型广场和返回错误为准。
</Note>

## 超时与重试

- 不要把视频生成当作普通同步请求长时间挂起。
- 如果返回 `task_id`，按任务接口轮询。
- 轮询间隔建议从 3 到 5 秒开始，避免触发 [429 限流](/errors/429)。
- 单次任务过久或中断，参考 [524 超时](/errors/524)。

## 成本控制

视频通常按次、时长、分辨率或模型倍率计费。上线前先用低规格参数测试，并在控制台日志查看实际扣费。详见 [计费与用量](/billing/pricing)。
