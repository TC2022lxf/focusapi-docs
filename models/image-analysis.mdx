图片分析是“模型看图并回答问题”的能力，适合截图理解、票据识别、图表分析、产品图质检、图片问答等场景。它和 [图像生成](/models/image-generation) 不同：本页是理解已有图片，图像生成是创建新图片。

## 常用接口

多数图片分析模型仍使用文本对话接口：

```http
POST /v1/chat/completions
```

区别在于 `messages.content` 不再只是字符串，而是文本和图片组成的数组。

## 支持的模型来源

| 来源 | 适合场景 | 说明 |
| --- | --- | --- |
| GPT 视觉模型 | 通用图片问答、截图理解、图表分析 | 通常兼容 OpenAI 图文输入格式 |
| Claude 视觉模型 | 长上下文、文档截图、复杂图片描述 | 可能支持 OpenAI 兼容方式或 Anthropic 原生格式 |
| Gemini 视觉模型 | 多模态理解、图片/文件综合分析 | 特殊文件参数以模型广场说明为准 |
| 其他视觉模型 | 垂直识别、低价批量分析 | 以模型广场能力标签为准 |

## curl 调用

```bash
curl https://api.focusapi.cn/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o-mini",
    "messages": [
      {
        "role": "user",
        "content": [
          { "type": "text", "text": "请描述这张图片的主要内容" },
          {
            "type": "image_url",
            "image_url": {
              "url": "https://example.com/image.png"
            }
          }
        ]
      }
    ]
  }'
```

<CodeGroup>

```python OpenAI SDK
from openai import OpenAI

client = OpenAI(
    base_url="https://api.focusapi.cn/v1",
    api_key="sk-your-api-key",
)

response = client.chat.completions.create(
    model="gpt-4o-mini",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "请描述这张图片的主要内容"},
                {
                    "type": "image_url",
                    "image_url": {
                        "url": "https://example.com/image.png",
                    },
                },
            ],
        }
    ],
)

print(response.choices[0].message.content)
```

```python Claude SDK
from anthropic import Anthropic

client = Anthropic(
    api_key="sk-your-api-key",
    base_url="https://api.focusapi.cn",
)

message = client.messages.create(
    model="claude-sonnet-4-20250514",
    max_tokens=1024,
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "请描述这张图片的主要内容"},
                {
                    "type": "image",
                    "source": {
                        "type": "url",
                        "url": "https://example.com/image.png",
                    },
                },
            ],
        }
    ],
)

print(message.content[0].text)
```

```python Gemini SDK
import requests
from google import genai
from google.genai import types

client = genai.Client(
    api_key="sk-your-api-key",
    http_options=types.HttpOptions(base_url="https://api.focusapi.cn"),
)

image_bytes = requests.get("https://example.com/image.png").content

response = client.models.generate_content(
    model="gemini-2.0-flash",
    contents=[
        "请描述这张图片的主要内容",
        types.Part.from_bytes(
            data=image_bytes,
            mime_type="image/png",
        ),
    ],
)

print(response.text)
```

```python LiteLLM
import litellm

response = litellm.completion(
    model="openai/gpt-4o-mini",
    api_key="sk-your-api-key",
    api_base="https://api.focusapi.cn/v1",
    messages=[
        {
            "role": "user",
            "content": [
                {"type": "text", "text": "请描述这张图片的主要内容"},
                {
                    "type": "image_url",
                    "image_url": {
                        "url": "https://example.com/image.png",
                    },
                },
            ],
        }
    ],
)

print(response.choices[0].message.content)
```

<Note>
  OpenAI SDK / LiteLLM 使用 OpenAI 兼容图文格式；Claude SDK 和 Gemini SDK 使用各自原生图片输入格式。若特殊格式报错，先用 curl 或 OpenAI SDK 验证模型是否支持视觉能力。
</Note>

## 图片输入建议

- 优先使用可公网访问的 HTTPS 图片 URL。
- base64 图片会显著增加请求体大小，超大图片容易触发 [524 超时](/errors/524)。
- 截图、票据、表格类图片要保证文字清晰。
- 批量分析时控制并发，避免 [429 限流](/errors/429)。

## 常见问题

| 问题 | 处理 |
| --- | --- |
| 模型说看不到图片 | 确认模型广场标记支持视觉能力 |
| 图片 URL 读取失败 | 确认 URL 可公网访问，且没有登录、Referer 或防盗链限制 |
| 返回内容不稳定 | 提示词写清输出格式和关注点 |
| 请求体太大 | 压缩图片或改用 URL，不要传超大 base64 |
