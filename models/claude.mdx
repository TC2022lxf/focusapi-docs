## 调用方式

FocusAPI 支持 **Anthropic Messages** 原生路径，也支持通过 **OpenAI 兼容** 的 `chat/completions` 调用部分 Claude 模型（以模型广场为准）。

### 方式 A：OpenAI 兼容（上手最快）

与 [OpenAI 格式](/models/openai) 相同，仅修改 `model` 为 Claude 系列 ID，例如：

```bash
curl https://api.FocusAPI.com/v1/chat/completions \
  -H "Authorization: Bearer sk-your-api-key" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "claude-sonnet-4-20250514",
    "messages": [{"role": "user", "content": "Hello"}]
  }'
```

### 方式 B：Anthropic Messages API

使用 Anthropic SDK 或 HTTP 时，将 Base URL 指向 FocusAPI 提供的 Claude 路径前缀（以控制台文档或公告为准）。常见路径形态为 `/v1/messages`（Anthropic 格式）。

请求头通常需要：

```http
x-api-key: sk-your-api-key
anthropic-version: 2023-06-01
```

<Note>
  若你已在用 OpenAI SDK，优先使用方式 A，减少集成成本。
</Note>

## 模型选择

在控制台 **模型广场** 筛选 Claude 系列，使用列表中的 **完整模型 ID**。

## 常见问题

- **401**：检查 `Authorization` 或 `x-api-key` 是否与令牌类型匹配，见 [401](/errors/401)。
- **扩展思考 / 工具**：高级参数是否支持取决于平台上架配置，失败时先简化请求体再试。
