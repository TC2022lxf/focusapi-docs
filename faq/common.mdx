## 接入与调用

<AccordionGroup>
  <Accordion title="和 OpenAI 官方 API 有什么关系？">
    FocusAPI 提供 **OpenAI 兼容** 的 HTTP 接口。你只需把 SDK 的 `base_url` 改为 `https://www.focusapi.cn/v1`，并使用 FocusAPI 颁发的 API Key。模型由平台聚合上架，非 OpenAI 官方账户直连。
  </Accordion>
  <Accordion title="Base URL 应该填什么？">
    默认：`https://www.focusapi.cn/v1`。若控制台提供了区域地址，以控制台为准。步骤见 [快速开始](/quickstart)。
  </Accordion>
  <Accordion title="模型名从哪里查？">
    登录控制台 → **模型广场**，复制完整模型 ID 到请求体 `model` 字段。
  </Accordion>
  <Accordion title="是否支持流式输出？">
    支持。在文本对话请求中设置 `"stream": true`，详见 [文本对话](/models/chat#流式输出)。
  </Accordion>
</AccordionGroup>

## 产品与定价

<AccordionGroup>
  <Accordion title="为什么比官网便宜？">
    FocusAPI 是 **AI 模型 API 聚合平台**，不是某一家厂商的官方直营店。平台通过以下方式把成本压下来，再让利给用户：

    - **多渠道智能路由**：同一模型可对接多条上游线路（共享配额池、批量采购、中转专线等），按价格与负载自动择优转发。
    - **资源复用与规模效应**：多用户共享上游容量，摊薄固定成本；平台侧统一网关、计费与监控，减少接入与运维开销。
    - **分级定价**：模型广场中不同价位对应不同线路质量，低价档走性价比通道，高价档走更稳定、更接近官方的通道。

    <Warning>
      **低价档的取舍**：更便宜通常意味着 **共享资源、更严格的限流、偶发排队或超时**。适合开发调试、原型验证、对 SLA 要求不高的场景。若业务对稳定性、延迟、并发敏感，请优先选择标注为 **原价** 或更高档位的模型。详见 [429 限流](/errors/429)、[524 超时](/errors/524)。
    </Warning>
  </Accordion>
  <Accordion title="为什么最贵的档位也叫「原价」？和官方一样吗？">
    模型广场里 **最贵的一档通常标注为「原价」**，含义是：按 **厂商官方 API 美元标价** 计费，不经大幅加价。

    具体换算方式：

    1. 取上游官方公布的 **美元单价**（按 token 或按次）。
    2. 乘以该模型的 **模型倍率**、**补全倍率** 等平台计费参数（见 [计费与价格](/billing/pricing)）。
    3. 再按平台 **默认汇率** 折算为人民币扣费。当前默认汇率为 **1 USD = 7.3 CNY**（如有调整，以控制台与模型广场实时展示为准）。

    「原价」档通常走 **直连或近官方质量的专用上游**，延迟与稳定性优于低价共享通道。平台主要利润来自 **汇率折算与规模运营**，而非在 token 单价上翻倍宰客。

    <Note>
      不同模型的「原价」美元基数不同；同一模型在不同分组/档位下的倍率也可能不同。下单前请以 **模型广场** 展示的参考价格为准。
    </Note>
  </Accordion>
  <Accordion title="用的是官方真模型吗？会不会有山寨或套壳？">
    **不会故意用假模型冒充官方。** FocusAPI 的定位是正规 API 聚合与分发：你请求的 `model` 会路由到配置好的 **真实上游 API**，返回的是该厂商对应模型的能力，而不是本地小模型套壳。

    我们的商业策略是 **薄利多销**：

    - 靠 **走量、路由效率、多渠道议价** 赚钱，而不是单次请求暴利。
    - 低价档与原价档的区别在于 **走哪条上游线路、共享还是独享**，不在于「是不是真模型」。
    - 平台会持续审计渠道质量；若某条线路异常，运营侧会降级或下线，而不是悄悄换成假货。

    若你对输出质量有疑虑，建议：① 先用 **原价档** 与官方 API 对比；② 在控制台 **日志** 中核对用量与扣费；③ 仍有疑问时通过控制台联系客服，说明模型名、时间与请求 ID。
  </Accordion>
  <Accordion title="不同价位的模型该怎么选？">

    | 场景 | 建议 |
    | --- | --- |
    | 本地开发、联调、Demo | 选 **低价 / 性价比** 档，控制成本 |
    | 生产环境、客服、代码助手 | 选 **原价** 或更高稳定档位 |
    | 长上下文、批量任务 | 注意限流，设置重试与 `max_tokens` 上限 |
    | 图像 / 视频等按次计费 | 先看模型广场单次参考价，小流量试跑 |

    更多选型思路见 [模型选择指南](/models/selection)。
  </Accordion>
  <Accordion title="便宜档不稳定、经常 429，是平台故意限流吗？">
    不完全是「故意卡你」，更多是 **上游共享容量饱和** 时的保护机制：

    - 低价通道往往多人共用同一批上游 Key 或配额池，高峰时段容易触发 **429 请求过于频繁**。
    - 偶发 **524 超时** 可能与上游排队、跨境链路、模型本身响应慢有关。
    - 平台会在网关层做 **公平限流**，避免少数用户占满全部共享配额。

    缓解办法：降低并发、指数退避重试、换 **原价档** 或更高分组、错峰调用。详见 [429 限流](/errors/429)、[524 超时](/errors/524)。
  </Accordion>
</AccordionGroup>

## 账户与计费

<AccordionGroup>
  <Accordion title="额度不足怎么办？">
    在控制台 **充值** 或使用 **兑换码**。扣费规则见 [计费与价格](/billing/pricing)。
  </Accordion>
  <Accordion title="如何查看某次请求花了多少？">
    在控制台 **日志** 查看历史记录；响应 JSON 中的 `usage` 字段也可参考。
  </Accordion>
  <Accordion title="汇率 7.3 是什么意思？会变动吗？">
    平台按 **1 美元 = 7.3 人民币** 的默认汇率，将官方美元标价折算为账户扣费金额（充值额度同理）。该汇率用于统一定价与展示，**可能随市场与运营策略调整**；充值页、模型广场标注的价格始终以 **实时展示** 为准。

    「原价」模型 = 官方美元单价 × 计费倍率 × 汇率，而不是随意定价。更多说明见本页 **「为什么最贵的档位也叫「原价」？」** 一节。
  </Accordion>
</AccordionGroup>

## 还不确定？

1. 先完成 [快速开始](/quickstart) 中的 curl 测试
2. 若 curl 成功而 SDK 失败，对比 `base_url` 与 `api_key` 是否一致
3. 仍无法定位时，从 [错误排查总览](/errors/overview) 按现象继续查
